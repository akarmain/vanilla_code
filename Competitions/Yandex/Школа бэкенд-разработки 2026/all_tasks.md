# Сборник задач

## 1. Подсказки при вводе

### Ограничения

| Параметр | Значение |
|---|---|
| Язык | Все языки |
| Время | 1 с |
| Память | 256 Мб |
| Ввод | стандартный ввод или `input.txt` |
| Вывод | стандартный вывод или `output.txt` |

### Условие

Вася придумал новую систему подсказок при вводе.

В этой системе заранее подготовлен список слов, которые могут быть использованы как подсказки. Пользователь вводит буквы, и для каждой буквы должно выбираться слово по следующим правилам:

1. Из словаря выбираются слова, начинающиеся с той же буквы, которую ввел пользователь.
2. Из выбранных слов выбираются те, которые использовались в качестве подсказки наименьшее количество раз.
3. Из выбранных слов выбирается лексикографически минимальное слово.

Это слово и является подсказкой.

Необходимо реализовать систему, придуманную Васей.

### Формат ввода

В первой строке содержится два числа `n` и `k`:

```text
1 ≤ n, k ≤ 100000
```

где:

- `n` — размер словаря;
- `k` — количество введенных пользователем букв.

В следующих `n` строках записаны слова словаря, по одному в строке.

Слова состоят из строчных английских букв, их длина не превосходит `21`. Все слова в словаре различны.

В следующих `k` строках записаны введенные пользователем буквы, по одной в строке.

Гарантируется, что для каждой вводимой буквы в словаре существует хотя бы одно слово, начинающееся с этой буквы.

### Формат вывода

Если введено некорректное значение `n` или `k`, выведите:

```text
-2
```

Иначе для каждой введённой пользователем буквы выведите слово-подсказку.

### Пример 1

#### Ввод

```text
4 5
peterburg
murmansk
perm
moscow
p
m
m
p
p
```

#### Вывод

```text
perm
moscow
murmansk
peterburg
perm
```

### Пример 2

#### Ввод

```text
5 3
int
main
void
double
string
v
m
v
```

#### Вывод

```text
void
main
void
```

### Пример 3

#### Ввод

```text
1 3
python
p
p
p
```

#### Вывод

```text
python
python
python
```

## 2. Значения и переменные

### Ограничения

| Параметр | Значение |
|---|---|
| Язык | Все языки |
| Время | 1 с |
| Память | 256 Мб |
| Ввод | стандартный ввод или `input.txt` |
| Вывод | стандартный вывод или `output.txt` |

### Условие

Даны две последовательности, состоящие из `n` элементов.

Каждый элемент может быть числом или названием переменной.

Необходимо определить, существует ли способ заменить каждую переменную на какое-либо число так, чтобы последовательности совпадали.

### Формат ввода

В первой строке вводится число `n`:

```text
1 ≤ n ≤ 50000
```

где `n` — длина последовательностей.

В следующих двух строках вводятся последовательности.

Каждая последовательность состоит из `n` элементов, разделенных одним пробелом.

Каждый элемент может быть:

- целым числом от `1` до `1000`;
- названием переменной, состоящим только из английских букв.

Длина названия переменной не превосходит `10`.

### Формат вывода

Если введено некорректное значение `n`, выведите:

```text
-2
```

Иначе выведите:

```text
YES
```

если существует способ замены переменных на числа так, чтобы последовательности совпадали.

В противном случае выведите:

```text
NO
```

### Пример 1

#### Ввод

```text
3
3 1 2
3 1 3
```

#### Вывод

```text
NO
```

### Пример 2

#### Ввод

```text
4
4 5 igrek igrek
4 iks 3 iks
```

#### Вывод

```text
NO
```

### Пример 3

#### Ввод

```text
5
x 3 x y 3
x y 2 z 3
```

#### Вывод

```text
YES
```

## 3. Покрытие прямоугольниками

### Ограничения

| Параметр | Значение |
|---|---|
| Язык | Все языки |
| Время | 1 с |
| Память | 256 Мб |
| Ввод | стандартный ввод или `input.txt` |
| Вывод | стандартный вывод или `output.txt` |

### Условие

На плоскости заданы `n` точек.

Каждую из них необходимо покрыть одним или несколькими прямоугольниками.

Точка считается покрытой многоугольником, если она лежит внутри него или на его границе.

Стороны каждого прямоугольника должны быть параллельны осям координат.

Центр каждого прямоугольника должен располагаться в точке `(0, 0)`.

Необходимо выбрать набор прямоугольников, покрывающих все точки, с минимальной суммарной площадью.

### Формат ввода

В первой строке вводится целое число `n`:

```text
1 ≤ n ≤ 5000
```

где `n` — количество точек.

В следующих `n` строках вводятся два целых числа `x` и `y`:

```text
|x|, |y| ≤ 5 ⋅ 10^7
x ≠ 0
y ≠ 0
```

### Формат вывода

Если введено некорректное значение `n` или координат точки, выведите:

```text
-2
```

Иначе выведите минимальную суммарную площадь прямоугольников, покрывающих все точки.

### Пример 1

#### Ввод

```text
2
1 1
-1 -1
```

#### Вывод

```text
4
```

### Пример 2

#### Ввод

```text
3
-7 19
9 -30
25 10
```

#### Вывод

```text
2080
```

### Пример 3

#### Ввод

```text
6
1 20
3 17
5 15
8 12
9 11
10 10
```

#### Вывод

```text
760
```

## 4. Пары шаблонов

### Ограничения

| Параметр | Значение |
|---|---|
| Язык | Все языки |
| Время | 1 с |
| Память | 256 Мб |
| Ввод | стандартный ввод или `input.txt` |
| Вывод | стандартный вывод или `output.txt` |

### Условие

Назовем шаблоном строку, состоящую из маленьких английских букв и вопросительных знаков (`?`).

Слово соответствует шаблону, если можно заменить вопросительные знаки в шаблоне на буквы таким образом, чтобы получилось слово.

Например, слово `baca` соответствует шаблонам `bac?` и `b??a`, но не соответствует шаблону `?b??`.

Дано `n` шаблонов одинаковой длины `m`.

Необходимо определить количество пар шаблонов `(i, j)` из этого набора, таких, что существует хотя бы одно слово, соответствующее обоим шаблонам, и выполняется условие:

```text
1 ≤ i < j ≤ n
```

### Формат ввода

В первой строке записаны числа `n` и `m`:

```text
1 ≤ n ≤ 50000
1 ≤ m ≤ 6
```

где:

- `n` — количество шаблонов;
- `m` — длина шаблонов.

В следующих `n` строках записаны шаблоны.

Каждый шаблон состоит из маленьких английских букв и вопросительных знаков и имеет длину `m`.

### Формат вывода

Если введено некорректное значение `n`, выведите:

```text
-2
```

Иначе выведите одно число — количество пар шаблонов, для которых найдется слово, соответствующее обоим шаблонам.

### Пример 1

#### Ввод

```text
3 3
??b
c??
c?c
```

#### Вывод

```text
2
```

### Пример 2

#### Ввод

```text
4 6
ab??c?
??kll?
a?k??c
?bcd??
```

#### Вывод

```text
3
```

### Пример 3

#### Ввод

```text
5 2
??
b?
c?
?g
cg
```

#### Вывод

```text
8
```

## 5. Архитектурный план

### Ограничения

| Параметр | Значение |
|---|---|
| Язык | Все языки |
| Время | 3 с |
| Память | 512 Мб |
| Ввод | стандартный ввод или `input.txt` |
| Вывод | стандартный вывод или `output.txt` |

### Условие

На центральном проспекте города было решено построить новый дом из блоков.

Каждый блок представляет собой клетку `1 × 1`.

В соответствии с архитектурным планом дом должен был быть переменной высоты: его можно представить в виде последовательности столбцов различной высоты и шириной в один блок.

Столбцы занумерованы числами от `1` до `n`.

Однако из-за обильных снегопадов в последние зимы в городе было введено новое правило строительства домов.

Теперь ровно один из столбцов в доме должен быть самым высоким, а остальные столбцы должны образовывать скат, чтобы снег не накапливался на крыше.

Если самым высоким столбцом является столбец `i` и его высота равна `h_i`, то высота столбца `j` должна быть равна:

```text
h_i - |i - j|
```

То есть соседние с `i` столбцы должны быть на `1` ниже `i`-го, столбцы с номерами `i - 2` и `i + 2` — на `2` ниже и так далее.

За одну операцию можно добавить или удалить один блок в любой из столбцов.

Необходимо изменить план дома, чтобы он соответствовал новому правилу, используя минимальное число операций.

После изменения высоты всех столбцов должны быть строго положительными. Если самым высоким является столбец `i`, то должно выполняться условие:

```text
h_i ≥ 1 + max(i - 1, n - i)
```

### Формат ввода

В первой строке вводится число `n`:

```text
1 ≤ n ≤ 100000
```

где `n` — количество столбцов в описании дома.

Во второй строке вводится `n` целых чисел `h_i`:

```text
1 ≤ h_i ≤ 10^9
```

где `h_i` — высота `i`-го столбца в изначальном архитектурном плане.

### Формат вывода

Если введено некорректное значение `n`, выведите:

```text
-2
```

Иначе выведите минимальное количество операций, необходимых для изменения плана дома в соответствии с новым правилом.

### Пример 1

#### Ввод

```text
4
1 1 2 3
```

#### Вывод

```text
3
```

### Пример 2

#### Ввод

```text
5
4 5 7 2 2
```

#### Вывод

```text
4
```

### Пример 3

#### Ввод

```text
6
4 5 6 5 4 3
```

#### Вывод

```text
0
```
